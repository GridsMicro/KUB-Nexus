# 💬 บันทึกการสนทนาและการตอบคำถาม (Q&A Logs)

หน้านี้รวบรวมคำถามที่เกิดขึ้นระหว่างการพัฒนาโครงการ **KUB-Nexus** เพื่อเป็นความรู้และแนวทางสำหรับอนาคต

---

### ❓ คำถาม: ซื้อขาย Bitcoin ผ่าน Bitkub บนเว็บได้ไหม?
**💡 คำตอบ:** ทำได้แน่นอนครับ! ผ่าน 2 วิธีหลัก:
1. **Bitkub Exchange API v3:** สำหรับการเทรดอัตโนมัติ (Autonomous) ผ่านโค้ด 
2. **Next.js Interface:** สร้าง Dashboard ส่วนตัวที่ดึงข้อมูลผ่าน Websocket และส่งคำสั่งเทรดผ่าน Server Actions ที่ปลอดภัย

---

### ❓ คำถาม: ถ้าคนอื่นจะใช้งานบ้าง เขาจะใส่ API เองยังไง ผู้ใช้ต้องเป็น Dev หรือไม่?
**💡 คำตอบ:** ไม่จำเป็นต้องเป็น Dev ครับ! เราออกแบบให้ใช้งานง่ายได้ 2 ทาง:
1. **Web Settings UI:** หน้าเว็บกรอก Key ที่สวยงามและซ่อนความลับ (Masking)
2. **IoT WiFi Portal:** อุปกรณ์ ESP32 จะปล่อย WiFi ให้เราเชื่อมต่อผ่านมือถือเพื่อกรอก API Key และรหัส WiFi บ้าน (Captive Portal) เหมือนเครื่องใช้ไฟฟ้าอัจฉริยะทั่วไป
3. **อนาคต (Bitkub NEXT):** การล็อกอินด้วย OAuth เพื่อให้ระบบขอสิทธิ์เข้าถึงโดยที่ผู้ใช้ไม่ต้องถือ Key เลย

---

### ❓ คำถาม: จะวาง API Key อย่างไรให้ปลอดภัย?
**💡 คำตอบ:** 
- **ฝั่ง Web:** ใช้ไฟล์ `.env.local` สำหรับการพัฒนา และรันบน Server-side เท่านั้น (ห้ามโชว์ที่ Client-side)
- **ฝั่ง Firmware:** ใช้ **ConfigManager (NVS)** ซึ่งเป็นหน่วยความจำถาวรภายในชิป ESP32 ที่เข้าถึงได้ยากจากภายนอก และห้ามเขียน Key ลงในซอร์สโค้ด (Hardcoding) โดยเด็ดขาด

---

### ❓ คำถาม: ผู้ใช้ต้องมีบัญชี Bitkub ไหม และ ล็อกอินยังไง?
**💡 คำตอบ:** 
- **จำเป็นต้องมีบัญชี Bitkub Exchange:** เพราะธุรกรรมจริงจะเกิดขึ้นในบัญชีของผู้ใช้เอง
- **วิธีการเชื่อมต่อ (ล็อกอิน):** 
    1. **API Key Method:** ผู้ใช้นำ API Key/Secret มากรอกในหน้า Settings ของแอป เหมาะสำหรับระบบอัตโนมัติที่ทำงานได้ 24 ชม. 
    2. **Bitkub NEXT:** ล็อกอินด้วย OAuth ผ่านเบอร์โทรศัพท์และ OTP เพื่อความง่ายและพรีเมียม
